# 18e: LLM Judge

## Overview

Add GPT-5.4 as a cross-model quality evaluator (LLM-as-judge) for content generated by Claude. Three evaluation dimensions: naturalness, brief adherence, and heading structure. Wrap all quality tiers into a single orchestrator (`quality_pipeline.py`) that replaces direct `run_quality_checks()` calls across the codebase.

Feature-flagged OFF by default (`QUALITY_TIER2_ENABLED=false`). When disabled, the pipeline returns identical results to the current system -- zero behavioral change.

**Cost:** ~$0.035/article at GPT-5.4 pricing ($2.50/1M input, $20/1M output). At 50 articles/month = ~$1.75/month.

**Latency:** ~2-4s added when Tier 2 enabled (3 parallel OpenAI calls).

---

## Decisions (from Planner/Advocate Debate)

### 1. httpx Direct Calls vs. OpenAI SDK

**Planner says:** Use raw httpx. We already have the pattern in `claude.py` -- 580 lines of httpx with circuit breaker, retry, rate limit handling. No new dependency.

**Advocate says:** The OpenAI SDK handles retries, rate limits, streaming, structured outputs, and model fallback out of the box. The `openai` package has only 4 dependencies (httpx, pydantic, sniffio, typing-extensions) -- all of which we already have. Reimplementing retry logic for a second LLM vendor is wasted effort and a maintenance burden.

**Decision: Use the `openai` SDK.** Rationale:
- We already have httpx and pydantic as deps, so `openai` adds near-zero transitive weight.
- The SDK handles streaming, structured outputs, retry-after headers, and API versioning -- things we'd have to maintain manually otherwise.
- This is a secondary vendor (judge only), not our primary generation path. We don't need the deep observability/circuit-breaker apparatus from `claude.py`. If OpenAI is down, we gracefully degrade to Tier 1 only.
- The SDK's `response_format={"type": "json_object"}` guarantees valid JSON responses, eliminating our biggest parsing headache.
- Add `openai>=1.30.0` to `pyproject.toml` dependencies and `openai` to the mypy overrides.

### 2. Is 15s Timeout Enough for GPT-5.4?

**Advocate says:** GPT-5.4 latency varies. On cold starts or under load, 15s may not be enough. A timeout that fires 5% of the time wastes money on partial calls.

**Decision: 30s default timeout, configurable via `QUALITY_TIER2_TIMEOUT=30`.** Rationale:
- Our content articles are 1000-2500 words. The judge prompt + content is ~3000-6000 tokens input, ~200 tokens output. GPT-5.4 should respond in 2-5s normally.
- 30s gives 6x headroom for cold starts and congestion.
- Since we run 3 calls in parallel, the wall-clock impact is max(call1, call2, call3), not sum.
- If it still times out at 30s, the graceful degradation path catches it and returns Tier 1 results only.

### 3. Scoring Formula -- Why -5 per Tier 1 Error?

**Advocate says:** The formula is arbitrary. A single banned word (`-5`) and a tier1 AI word (`-5`) get the same deduction, but a banned word is far more serious -- it could be offensive. Should we weight by severity?

**Decision: Severity-weighted deductions with documented rationale.** Final formula:

```
Start at 100

Tier 1 deductions (deterministic, high confidence):
  - tier1_ai_word:        -5 each  (definite AI tell, must fix)
  - banned_word:          -8 each  (client-specific prohibition)
  - competitor_name:      -8 each  (never mention competitors)
  - em_dash:              -2 each  (style issue, not critical)
  - ai_pattern:           -5 each  (AI opener, definite tell)
  - triplet_excess:       -3 total (per-piece, not per-instance)
  - rhetorical_excess:    -3 total (per-piece, not per-instance)
  - tier2_ai_excess:      -3 total (per-piece, not per-instance)
  - negation_contrast:    -3 total (per-piece, not per-instance)

Tier 1b deductions (bible checks):
  - bible_preferred_term: -5 each  (wrong terminology)
  - bible_banned_claim:   -8 each  (factually wrong)
  - bible_wrong_attribution: -5 each
  - bible_term_context:   -5 each

Blog-specific deductions:
  - tier3_banned_phrase:  -5 each
  - empty_signpost:       -3 each
  - missing_direct_answer: -5 each
  - business_jargon:      -3 each

Tier 2 deductions (LLM judge, advisory):
  - naturalness < threshold:       -15
  - brief_adherence < threshold:   -10
  - heading_structure < 0.5:       -5

Floor at 0, cap at 100.
```

**Why these weights:** The formula is intentionally conservative. Deterministic issues (Tier 1) get moderate per-item deductions because they're fixable. The LLM judge gets larger single deductions because a low naturalness score means the whole piece has a systemic problem. Banned words and competitor names get the heaviest per-item deductions because they're the most operationally damaging.

The formula is centralized in one place (`quality_pipeline.py::_compute_score`) and trivially tunable. We can adjust weights based on real-world data after launch.

### 4. Short-Circuit Saves $0.035 -- Is It Worth the Complexity?

**Advocate says:** Short-circuiting adds ~15 lines of code. The savings per article is $0.035. If we process 50 articles with 10% critical failure rate, we save $0.175. Is the conditional branch worth the cognitive load?

**Decision: Keep short-circuiting, but keep it simple.** Rationale:
- It's not about the $0.035. It's about not wasting 3s of latency evaluating content that definitely needs fixing. If the content literally says "delve" or makes a banned claim, the operator is going to fix it regardless of what GPT-5.4 thinks about naturalness.
- Implementation is trivial: check if any issue has `type in CRITICAL_ISSUE_TYPES` before calling `run_llm_judge_checks()`. One `if` statement.
- The critical types are: `tier1_ai_word`, `banned_word`, `competitor_name`, `bible_banned_claim`.

### 5. What If OpenAI Changes Their API Response Format?

**Decision: Use the SDK.** The `openai` SDK abstracts API versioning. When OpenAI changes formats, the SDK maintainers handle it. This is the primary reason to use the SDK over raw httpx for a secondary vendor.

### 6. Should We Cache LLM Judge Results?

**Advocate says:** If an operator hits "Recheck" after editing content, and the content didn't change in the fields that matter, we'd be paying $0.035 for the same result.

**Decision: No caching in v1.** Rationale:
- Recheck is infrequent (operators edit then recheck, so content IS different).
- Content hash comparison adds complexity for a rare case.
- The content brief may have changed between checks (new LSI terms from POP refresh).
- If we see unnecessary repeat calls in production, we can add a content-hash check as a fast follow.

### 7. Sequential or Parallel LLM Calls?

**Decision: Parallel with `asyncio.gather()`.** All 3 judge functions are independent. Running them in parallel reduces wall-clock time from ~6-9s to ~2-4s. Since we're using the async OpenAI client, this is trivial.

### 8. Rate Limiting for Batch Generation (50 pages = 150 calls)

**Advocate says:** 50 pages * 3 calls = 150 OpenAI calls. GPT-5.4 rate limits are generous but not infinite. If we hit rate limits, cascading retries could stall the pipeline.

**Decision: Use the OpenAI SDK's built-in retry (which respects `retry-after` headers) + add a semaphore in `quality_pipeline.py`.** The pipeline already uses a concurrency semaphore (`content_generation_concurrency`, default 1). Since Tier 2 runs inside the per-page pipeline, and we default to concurrency=1, we'll only have 3 parallel OpenAI calls at a time. In production with concurrency=5, that's 15 parallel calls -- well within rate limits. If we need to throttle further, we add a `quality_tier2_concurrency` setting.

### 9. Are the Rubric Prompts Good Enough?

**Advocate says:** The prompts are the most critical part. Bad rubrics = garbage scores. Have we tested these against real content?

**Decision: Ship with well-structured rubrics, iterate based on real data.** The prompts below are designed with:
- Clear numeric anchors (what does 0.3 vs 0.7 vs 0.9 mean?)
- Concrete examples at each score level
- JSON output format enforced via `response_format`
- Brief adherence includes the actual brief data, not just "does it follow the brief?"
- Heading structure checks SEO-specific requirements

We log all judge responses to `qa_results.tier2` including the raw reasoning. After 50 articles, we can audit the reasoning strings to see if the rubric needs tuning.

### 10. Should Brief Adherence Check LSI Terms?

**Decision: Yes, include top 10 LSI terms by weight in the brief adherence prompt.** The brief adherence check should verify:
- Primary keyword is addressed
- Target topics/themes from the brief are covered
- Top LSI terms appear naturally (not keyword-stuffed)
- Approximate word count target is met

This gives the judge concrete criteria, not just "does it follow the brief?"

"""LLM-as-judge quality evaluation using GPT-5.4.

Sends content to OpenAI's GPT-5.4 for three evaluation dimensions:
- Naturalness: Does the text read like a human expert wrote it?
- Brief Adherence: Does the content address the content brief?
- Heading Structure: Are headings logical, keyword-optimized, hierarchical?

Each function returns (score: float, reasoning: str) where score is 0.0-1.0.
Results are ADVISORY -- shown as warnings, never blocking.

Uses the openai SDK for built-in retry, rate limit handling, and JSON mode.
"""

import time

NATURALNESS_SYSTEM_PROMPT = """You are an expert editor evaluating whether text reads naturally or appears AI-generated.

You will receive a piece of web content (HTML). Evaluate how natural and human-written it sounds.

Score the text from 0.0 to 1.0 using these anchors:

- 0.9-1.0: Reads like a domain expert wrote it. Distinctive voice, varied sentence structure, natural flow. Could not distinguish from human-written content.
- 0.7-0.8: Mostly natural. Occasional formulaic transitions or generic phrasing, but overall reads well. A careful reader might suspect AI but wouldn't be sure.
- 0.5-0.6: Noticeably AI-generated. Formulaic structure, predictable transitions ("Furthermore...", "Additionally..."), lacks personality. An average reader would suspect AI.
- 0.3-0.4: Obviously AI-generated. Robotic cadence, excessive hedging, bullet-point-like prose, generic filler. Anyone would recognize this as AI.
- 0.0-0.2: Extreme AI slop. Contradictions, repetitive structure, no coherent voice, reads like a template was filled in.

Focus on:
- Sentence variety (length, structure, openers)
- Transition naturalness (not formulaic)
- Voice consistency (does it sound like one person?)
- Specificity (concrete details vs. vague generalities)
- Absence of AI tells (hedging, excessive qualifiers, "It's important to note...")

Respond with ONLY a JSON object:
{"score": <float 0.0-1.0>, "reasoning": "<2-3 sentences explaining your score>"}"""

BRIEF_ADHERENCE_SYSTEM_PROMPT = """You are an SEO content strategist evaluating whether a piece of content properly addresses its content brief.

You will receive:
1. The generated content (HTML)
2. The content brief (keyword, topics to cover, LSI terms, target word count)

Evaluate how well the content fulfills the brief requirements.

Score from 0.0 to 1.0 using these anchors:

- 0.9-1.0: Excellent adherence. Primary keyword naturally woven throughout. All required topics addressed with depth. LSI terms appear naturally. Word count within target range.
- 0.7-0.8: Good adherence. Primary keyword present and well-used. Most brief topics covered. Some LSI terms present. Minor gaps in coverage or slightly off word count.
- 0.5-0.6: Partial adherence. Primary keyword present but underused. Several brief topics missing or only superficially addressed. Few LSI terms. Content may have gone off on tangents.
- 0.3-0.4: Poor adherence. Content is loosely related to the brief but misses key requirements. Major topics absent. Feels like it was written for a different keyword.
- 0.0-0.2: Off-topic. Content does not address the brief. Wrong subject, wrong angle, or generic filler that could apply to any topic.

Focus on:
- Primary keyword usage (in H1, intro, throughout body -- natural, not stuffed)
- Topic coverage (are the brief's required topics addressed?)
- LSI term integration (do related terms appear naturally?)
- Word count alignment (within +/- 20% of target)
- Content depth (substantial coverage, not surface-level)

Respond with ONLY a JSON object:
{"score": <float 0.0-1.0>, "reasoning": "<2-3 sentences explaining your score, noting any missed brief points>"}"""

HEADING_STRUCTURE_SYSTEM_PROMPT = """You are an SEO specialist evaluating the heading structure of web content.

You will receive HTML content. Evaluate the heading hierarchy for SEO effectiveness and logical flow.

Score from 0.0 to 1.0 using these anchors:

- 0.9-1.0: Excellent structure. H1 contains primary keyword. H2s break content into logical sections that support the topic. H3s provide relevant sub-detail. Hierarchy is clean (no skipped levels). Headings form a scannable outline of the content.
- 0.7-0.8: Good structure. H1 present with keyword. H2s mostly logical. Minor issues: one heading could be more descriptive, or a section feels misplaced. Overall hierarchy works.
- 0.5-0.6: Adequate structure. H1 present but keyword usage is awkward. Some H2s are generic ("Key Benefits", "Things to Consider"). Hierarchy has minor violations. A reader could follow but it's not optimized.
- 0.3-0.4: Poor structure. H1 missing keyword or is generic. Headings don't form a logical outline. Skipped heading levels (H1 -> H3). Multiple generic headings. Content sections feel randomly ordered.
- 0.0-0.2: Broken structure. No clear H1, or multiple H1s. Headings don't relate to content beneath them. No logical flow. Would confuse both readers and search engines.

Focus on:
- H1: Contains primary keyword, accurately describes the page
- H2s: Break content into logical, scannable sections
- H3s: Provide meaningful sub-structure (not just decoration)
- Hierarchy: No skipped levels (H1 -> H2 -> H3, not H1 -> H3)
- Descriptiveness: Headings tell you what's in the section (not just "Benefits" but "Benefits of X for Y")
- Flow: Sections follow a logical progression a reader would expect

Respond with ONLY a JSON object:
{"score": <float 0.0-1.0>, "reasoning": "<2-3 sentences explaining your score>"}"""

# Per 1M tokens
_INPUT_COST_PER_M = 2.50
_OUTPUT_COST_PER_M = 20.00


def _estimate_cost(input_tokens: int, output_tokens: int) -> float:
    """Estimate USD cost for a GPT-5.4 call."""
    return (input_tokens * _INPUT_COST_PER_M / 1_000_000) + (
        output_tokens * _OUTPUT_COST_PER_M / 1_000_000
    )

def _parse_judge_response(raw_text: str, dimension: str) -> JudgeResult:
    """Parse LLM judge response JSON into a JudgeResult.

    Handles markdown code fences, extra whitespace, and malformed JSON
    gracefully -- returns a score of 0.0 with an error message on failure.
    """
    cleaned = raw_text.strip()

    # Strip markdown code fences if present
    if cleaned.startswith("```"):
        lines = cleaned.split("\n")
        lines = lines[1:]  # Remove opening fence
        if lines and lines[-1].strip() == "```":
            lines = lines[:-1]
        cleaned = "\n".join(lines).strip()

    # Try to extract JSON object if surrounded by text
    if not cleaned.startswith("{"):
        match = re.search(r"\{[^{}]*\}", cleaned)
        if match:
            cleaned = match.group(0)

    try:
        parsed = json.loads(cleaned)
    except json.JSONDecodeError as e:
        logger.warning(
            "LLM judge response JSON parse failed",
            extra={
                "dimension": dimension,
                "error": str(e),
                "raw_snippet": raw_text[:200],
            },
        )
        return JudgeResult(
            dimension=dimension,
            score=0.0,
            reasoning="",
            error=f"JSON parse failed: {e}",
        )

    score = parsed.get("score", 0.0)
    reasoning = parsed.get("reasoning", "")

    # Validate score is a number in range
    try:
        score = float(score)
        score = max(0.0, min(1.0, score))
    except (TypeError, ValueError):
        logger.warning(
            "LLM judge returned non-numeric score",
            extra={"dimension": dimension, "raw_score": score},
        )
        score = 0.0

    return JudgeResult(
        dimension=dimension,
        score=score,
        reasoning=str(reasoning),
    )

async def _call_openai(
    client: Any,
    system_prompt: str,
    user_prompt: str,
    model: str,
    timeout: float,
) -> tuple[str, int, int]:
    """Make a single OpenAI chat completion call.

    Returns (response_text, input_tokens, output_tokens).
    Raises on timeout or API error (caller handles).
    """
    response = await client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        response_format={"type": "json_object"},
        temperature=0.0,
        max_tokens=300,
        timeout=timeout,
    )

    text = response.choices[0].message.content or ""
    usage = response.usage
    input_tokens = usage.prompt_tokens if usage else 0
    output_tokens = usage.completion_tokens if usage else 0

    return text, input_tokens, output_tokens


async def score_naturalness(
    client: Any,
    content_html: str,
    model: str,
    timeout: float,
) -> JudgeResult:
    """Score content naturalness (0.0-1.0).

    Evaluates whether the text reads like a human expert wrote it
    or shows signs of AI generation.
    """
    # Truncate content to ~8000 chars to stay within reasonable token limits
    truncated = content_html[:8000]
    if len(content_html) > 8000:
        truncated += "\n\n[Content truncated for evaluation]"

    user_prompt = f"Evaluate the naturalness of this content:\n\n{truncated}"

    try:
        text, inp, out = await _call_openai(
            client, NATURALNESS_SYSTEM_PROMPT, user_prompt, model, timeout
        )
        result = _parse_judge_response(text, "naturalness")
        logger.debug(
            "Naturalness judge complete",
            extra={
                "score": result.score,
                "input_tokens": inp,
                "output_tokens": out,
            },
        )
        return result
    except Exception as e:
        logger.warning(
            "Naturalness judge failed",
            extra={"error": str(e), "error_type": type(e).__name__},
        )
        return JudgeResult(
            dimension="naturalness",
            score=0.0,
            reasoning="",
            error=str(e),
        )

def build_brief_summary(
    primary_keyword: str,
    content_brief: Any | None,
    target_word_count: int | None = None,
) -> str:
    """Build a human-readable brief summary for the brief adherence judge.

    Extracts keyword, LSI terms, related questions, and word count target
    from the content brief object (ORM model or SimpleNamespace).
    """
    parts: list[str] = []
    parts.append(f"Primary keyword: {primary_keyword}")

    if content_brief is not None:
        # LSI terms (top 10 by weight)
        lsi_terms = getattr(content_brief, "lsi_terms", None) or []
        if lsi_terms:
            # Sort by weight descending, take top 10
            sorted_terms = sorted(
                lsi_terms,
                key=lambda t: t.get("weight", 0) if isinstance(t, dict) else 0,
                reverse=True,
            )[:10]
            term_strs = []
            for t in sorted_terms:
                phrase = t.get("phrase", str(t)) if isinstance(t, dict) else str(t)
                term_strs.append(phrase)
            parts.append(f"LSI terms to include: {', '.join(term_strs)}")

        # Related questions
        related_q = getattr(content_brief, "related_questions", None) or []
        if related_q:
            parts.append(f"Related questions to address: {'; '.join(related_q[:5])}")

        # Word count
        wc_target = getattr(content_brief, "word_count_target", None)
        wc_min = getattr(content_brief, "word_count_min", None)
        wc_max = getattr(content_brief, "word_count_max", None)
        if wc_min and wc_max:
            parts.append(f"Target word count: {wc_min}-{wc_max}")
        elif wc_target:
            parts.append(f"Target word count: ~{wc_target}")

    if target_word_count and "Target word count" not in "\n".join(parts):
        parts.append(f"Target word count: ~{target_word_count}")

    return "\n".join(parts)


# ---------------------------------------------------------------------------
# Main entry point
# ---------------------------------------------------------------------------


async def run_llm_judge_checks(
    content_html: str,
    primary_keyword: str,
    content_brief: Any | None = None,
    target_word_count: int | None = None,
) -> JudgeRunResult:
    """Run all three LLM judge evaluations in parallel.

    Creates an AsyncOpenAI client, runs naturalness, brief adherence,
    and heading structure checks concurrently, and returns aggregated results.

    Returns JudgeRunResult with scores and any QualityIssues for scores
    below their thresholds.

    Graceful degradation: if OpenAI is unavailable or any call fails,
    returns partial results with error information. Never raises.
    """
    settings = get_settings()
    model = settings.quality_judge_model
    timeout = float(settings.quality_tier2_timeout)
    api_key = settings.openai_api_key

    if not api_key:
        return JudgeRunResult(
            issues=[],
            error="OpenAI API key not configured",
        )

    # Import here to handle missing dependency gracefully
    try:
        from openai import AsyncOpenAI
    except ImportError:
        logger.error("openai package not installed -- cannot run LLM judge")
        return JudgeRunResult(
            issues=[],
            error="openai package not installed",
        )

    client = AsyncOpenAI(api_key=api_key)
    brief_summary = build_brief_summary(
        primary_keyword, content_brief, target_word_count
    )

    start_time = time.monotonic()

    # Run all 3 evaluations in parallel
    naturalness_task = score_naturalness(client, content_html, model, timeout)
    adherence_task = score_brief_adherence(
        client, content_html, brief_summary, model, timeout
    )
    heading_task = score_heading_structure(
        client, content_html, primary_keyword, model, timeout
    )

    results = await asyncio.gather(
        naturalness_task,
        adherence_task,
        heading_task,
        return_exceptions=True,
    )

    latency_ms = (time.monotonic() - start_time) * 1000

    # Unpack results, handling any unexpected exceptions
    nat_result: JudgeResult | None = None
    adh_result: JudgeResult | None = None
    head_result: JudgeResult | None = None

    for i, r in enumerate(results):
        if isinstance(r, Exception):
            logger.warning(
                "LLM judge task raised unexpected exception",
                extra={"index": i, "error": str(r), "error_type": type(r).__name__},
            )
            continue
        if isinstance(r, JudgeResult):
            if r.dimension == "naturalness":
                nat_result = r
            elif r.dimension == "brief_adherence":
                adh_result = r
            elif r.dimension == "heading_structure":
                head_result = r

    # Build QualityIssues for scores below thresholds
    issues: list[QualityIssue] = []
    nat_threshold = settings.quality_naturalness_threshold
    adh_threshold = settings.quality_brief_adherence_threshold

    if nat_result and not nat_result.error and nat_result.score < nat_threshold:
        issues.append(
            QualityIssue(
                type="llm_naturalness",
                field="all_fields",
                description=(
                    f"Content naturalness score {nat_result.score:.2f} "
                    f"is below threshold {nat_threshold}"
                ),
                context=nat_result.reasoning[:200] if nat_result.reasoning else "",
            )
        )

    if adh_result and not adh_result.error and adh_result.score < adh_threshold:
        issues.append(
            QualityIssue(
                type="llm_brief_adherence",
                field="all_fields",
                description=(
                    f"Brief adherence score {adh_result.score:.2f} "
                    f"is below threshold {adh_threshold}"
                ),
                context=adh_result.reasoning[:200] if adh_result.reasoning else "",
            )
        )

    if head_result and not head_result.error and head_result.score < 0.5:
        issues.append(
            QualityIssue(
                type="llm_heading_structure",
                field="all_fields",
                description=(
                    f"Heading structure score {head_result.score:.2f} "
                    f"indicates structural issues"
                ),
                context=head_result.reasoning[:200] if head_result.reasoning else "",
            )
        )

    # Estimate total cost
    # Rough estimate: ~3000 input tokens per call, ~150 output tokens per call
    # Actual tokens logged per-call but not tracked aggregate here for simplicity
    estimated_cost = 3 * _estimate_cost(3000, 150)

    run_result = JudgeRunResult(
        issues=issues,
        naturalness=nat_result.score if nat_result and not nat_result.error else None,
        brief_adherence=(
            adh_result.score if adh_result and not adh_result.error else None
        ),
        heading_structure=(
            head_result.score if head_result and not head_result.error else None
        ),
        model=model,
        cost_usd=estimated_cost,
        latency_ms=latency_ms,
    )

    logger.info(
        "LLM judge run complete",
        extra={
            "naturalness": run_result.naturalness,
            "brief_adherence": run_result.brief_adherence,
            "heading_structure": run_result.heading_structure,
            "issues_count": len(issues),
            "latency_ms": round(latency_ms),
            "model": model,
        },
    )

    return run_result
```

## quality_pipeline.py -- Orchestrator

Location: `backend/app/services/quality_pipeline.py`

```python
"""Quality pipeline orchestrator.

Replaces direct run_quality_checks() calls with a multi-tier pipeline:
  Tier 1:  Deterministic regex/string checks (existing, free, <10ms)
  Tier 1b: Bible rule checks (if bibles matched, free, <10ms)
  Tier 2:  LLM judge via GPT-5.4 (optional, ~$0.035, ~2-4s)

Feature flags:
  QUALITY_TIER2_ENABLED=false  -- Tier 2 skipped, pipeline returns Tier 1 only
  QUALITY_TIER2_ENABLED=true   -- Full pipeline with LLM judge

Short-circuit: if critical Tier 1 failures exist (tier1_ai_word, banned_word,
competitor_name, bible_banned_claim), Tier 2 is skipped to save cost and latency.

Graceful degradation: if Tier 2 fails (timeout, API error, missing key),
the pipeline still returns Tier 1 results with a warning.
"""

import time
from dataclasses import dataclass, field
from datetime import UTC, datetime
from typing import Any

from app.core.config import get_settings
from app.core.logging import get_logger
from app.services.content_quality import (
    QualityIssue,
    QualityResult,
    run_blog_quality_checks,
    run_quality_checks,
)

logger = get_logger(__name__)


# Issue types that trigger short-circuit (skip Tier 2)
CRITICAL_ISSUE_TYPES = frozenset({
    "tier1_ai_word",
    "banned_word",
    "competitor_name",
    "bible_banned_claim",
})

# Score deductions by issue type
DEDUCTION_MAP: dict[str, int] = {
    # Tier 1: per-item deductions
    "tier1_ai_word": 5,
    "banned_word": 8,
    "competitor_name": 8,
    "em_dash": 2,
    "ai_pattern": 5,
    # Tier 1: per-piece deductions (applied once regardless of count)
    "triplet_excess": 3,
    "rhetorical_excess": 3,
    "tier2_ai_excess": 3,
    "negation_contrast": 3,
    # Tier 1b: bible checks
    "bible_preferred_term": 5,
    "bible_banned_claim": 8,
    "bible_wrong_attribution": 5,
    "bible_term_context": 5,
    # Blog-specific
    "tier3_banned_phrase": 5,
    "empty_signpost": 3,
    "missing_direct_answer": 5,
    "business_jargon": 3,
}

# Issue types where deduction is per-piece (not per-instance)
PER_PIECE_TYPES = frozenset({
    "triplet_excess",
    "rhetorical_excess",
    "tier2_ai_excess",
    "negation_contrast",
})

# Score tier labels and boundaries
SCORE_TIERS = [
    (90, "publish_ready"),
    (70, "minor_issues"),
    (50, "needs_attention"),
    (0, "needs_rewrite"),
]


@dataclass
class PipelineResult:
    """Complete result from the quality pipeline."""

    passed: bool
    score: int  # 0-100
    score_tier: str  # "publish_ready", "minor_issues", etc.
    issues: list[QualityIssue] = field(default_factory=list)
    checked_at: str = ""
    tier2: dict[str, Any] | None = None  # LLM judge metadata
    bibles_matched: list[str] = field(default_factory=list)
    short_circuited: bool = False

    def to_dict(self) -> dict[str, Any]:
        """Serialize to dict for storage in qa_results JSONB."""
        result: dict[str, Any] = {
            "passed": self.passed,
            "score": self.score,
            "score_tier": self.score_tier,
            "issues": [_issue_to_extended_dict(issue) for issue in self.issues],
            "checked_at": self.checked_at,
        }
        if self.tier2 is not None:
            result["tier2"] = self.tier2
        if self.bibles_matched:
            result["bibles_matched"] = self.bibles_matched
        if self.short_circuited:
            result["short_circuited"] = True
        return result


def _issue_to_extended_dict(issue: QualityIssue) -> dict[str, Any]:
    """Convert QualityIssue to dict with tier metadata."""
    d = issue.to_dict()
    # Add tier based on issue type
    if issue.type.startswith("llm_"):
        d["tier"] = 2
    elif issue.type.startswith("bible_"):
        d["tier"] = "1b"
    else:
        d["tier"] = 1
    return d


def _get_score_tier(score: int) -> str:
    """Map a score (0-100) to its tier label."""
    for threshold, label in SCORE_TIERS:
        if score >= threshold:
            return label
    return "needs_rewrite"


def _compute_score(
    issues: list[QualityIssue],
    naturalness: float | None = None,
    brief_adherence: float | None = None,
    heading_structure: float | None = None,
) -> int:
    """Compute composite quality score (0-100) from issues and LLM scores.

    Deterministic issues get per-item or per-piece deductions.
    LLM scores get threshold-based deductions.
    """
    settings = get_settings()
    score = 100

    # Track which per-piece types we've already deducted for
    deducted_per_piece: set[str] = set()

    for issue in issues:
        # Skip LLM issues -- they're handled separately below
        if issue.type.startswith("llm_"):
            continue

        deduction = DEDUCTION_MAP.get(issue.type, 3)  # default 3 for unknown

        if issue.type in PER_PIECE_TYPES:
            if issue.type not in deducted_per_piece:
                score -= deduction
                deducted_per_piece.add(issue.type)
        else:
            score -= deduction

    # LLM judge deductions (threshold-based)
    if naturalness is not None and naturalness < settings.quality_naturalness_threshold:
        score -= 15
    if (
        brief_adherence is not None
        and brief_adherence < settings.quality_brief_adherence_threshold
    ):
        score -= 10
    if heading_structure is not None and heading_structure < 0.5:
        score -= 5

    return max(0, min(100, score))


# ---------------------------------------------------------------------------
# Main pipeline
# ---------------------------------------------------------------------------


async def run_quality_pipeline(
    content: Any,
    brand_config: dict[str, Any],
    primary_keyword: str = "",
    content_brief: Any | None = None,
    is_blog: bool = False,
    fields: dict[str, str] | None = None,
) -> PipelineResult:
    """Run the full quality pipeline on content.

    This is the single entry point that replaces direct run_quality_checks()
    and run_blog_quality_checks() calls throughout the codebase.

    Args:
        content: PageContent ORM instance (for onboarding pages) or None
            (for blog posts where fields are passed directly).
        brand_config: BrandConfig.v2_schema dict.
        primary_keyword: The page's primary keyword (for Tier 2 brief check).
        content_brief: ContentBrief or SimpleNamespace with brief data.
        is_blog: If True, runs blog-specific checks (13 total).
        fields: Pre-extracted content fields dict. If None, extracted from
            the `content` PageContent instance. Required when is_blog=True.

    Returns:
        PipelineResult with score, issues, and optional Tier 2 metadata.
        Also stores result in content.qa_results if content is a PageContent.
    """
    settings = get_settings()
    start_time = time.monotonic()

    # -----------------------------------------------------------------------
    # STEP 1: Run Tier 1 deterministic checks
    # -----------------------------------------------------------------------
    if is_blog and fields is not None:
        tier1_result = run_blog_quality_checks(fields, brand_config)
    elif content is not None:
        tier1_result = run_quality_checks(content, brand_config)
    else:
        # Fallback: no content to check
        tier1_result = QualityResult(
            passed=True,
            issues=[],
            checked_at=datetime.now(UTC).isoformat(),
        )

    all_issues: list[QualityIssue] = list(tier1_result.issues)

    # -----------------------------------------------------------------------
    # STEP 2: Check feature flag for Tier 2
    # -----------------------------------------------------------------------
    tier2_enabled = settings.quality_tier2_enabled
    tier2_metadata: dict[str, Any] | None = None

    if not tier2_enabled:
        # Tier 2 disabled -- compute score from Tier 1 only
        score = _compute_score(all_issues)
        score_tier = _get_score_tier(score)

        result = PipelineResult(
            passed=len(all_issues) == 0,
            score=score,
            score_tier=score_tier,
            issues=all_issues,
            checked_at=datetime.now(UTC).isoformat(),
        )

        # Store in content.qa_results if applicable
        if content is not None and hasattr(content, "qa_results"):
            content.qa_results = result.to_dict()

        return result

    # -----------------------------------------------------------------------
    # STEP 3: Short-circuit check
    # -----------------------------------------------------------------------
    has_critical = any(
        issue.type in CRITICAL_ISSUE_TYPES for issue in all_issues
    )

    if has_critical:
        logger.info(
            "Quality pipeline short-circuiting: critical Tier 1 issues found",
            extra={
                "critical_types": [
                    i.type for i in all_issues if i.type in CRITICAL_ISSUE_TYPES
                ],
            },
        )

        score = _compute_score(all_issues)
        score_tier = _get_score_tier(score)

        result = PipelineResult(
            passed=False,
            score=score,
            score_tier=score_tier,
            issues=all_issues,
            checked_at=datetime.now(UTC).isoformat(),
            short_circuited=True,
        )

        if content is not None and hasattr(content, "qa_results"):
            content.qa_results = result.to_dict()

        return result

    # -----------------------------------------------------------------------
    # STEP 4: Run Tier 2 -- LLM Judge
    # -----------------------------------------------------------------------
    from app.services.llm_judge import run_llm_judge_checks

    # Build combined content HTML for the judge
    content_html = ""
    if fields:
        content_html = "\n\n".join(fields.values())
    elif content is not None:
        # Extract from PageContent fields
        for field_name in ("page_title", "meta_description", "top_description", "bottom_description"):
            val = getattr(content, field_name, None)
            if val:
                content_html += f"\n\n{val}"
        content_html = content_html.strip()

    # Get target word count for brief adherence
    target_wc = None
    if content_brief:
        target_wc = getattr(content_brief, "word_count_target", None)

    try:
        judge_result = await run_llm_judge_checks(
            content_html=content_html,
            primary_keyword=primary_keyword,
            content_brief=content_brief,
            target_word_count=target_wc,
        )

        # Merge LLM issues into all_issues
        all_issues.extend(judge_result.issues)

        tier2_metadata = {
            "model": judge_result.model,
            "naturalness": judge_result.naturalness,
            "brief_adherence": judge_result.brief_adherence,
            "heading_structure": judge_result.heading_structure,
            "cost_usd": judge_result.cost_usd,
            "latency_ms": round(judge_result.latency_ms),
        }

        if judge_result.error:
            tier2_metadata["error"] = judge_result.error

    except Exception as e:
        # Graceful degradation: Tier 2 failure doesn't break the pipeline
        logger.warning(
            "Tier 2 LLM judge failed, returning Tier 1 results only",
            extra={"error": str(e), "error_type": type(e).__name__},
            exc_info=True,
        )
        tier2_metadata = {
            "error": f"LLM judge failed: {e}",
            "model": settings.quality_judge_model,
        }
        judge_result = None

    # -----------------------------------------------------------------------
    # STEP 5: Compute final score
    # -----------------------------------------------------------------------
    nat_score = judge_result.naturalness if judge_result else None
    adh_score = judge_result.brief_adherence if judge_result else None
    head_score = judge_result.heading_structure if judge_result else None

    score = _compute_score(all_issues, nat_score, adh_score, head_score)
    score_tier = _get_score_tier(score)

    total_latency = (time.monotonic() - start_time) * 1000

    result = PipelineResult(
        passed=len(all_issues) == 0,
        score=score,
        score_tier=score_tier,
        issues=all_issues,
        checked_at=datetime.now(UTC).isoformat(),
        tier2=tier2_metadata,
    )

    # Store in content.qa_results if applicable
    if content is not None and hasattr(content, "qa_results"):
        content.qa_results = result.to_dict()

    logger.info(
        "Quality pipeline complete",
        extra={
            "score": score,
            "score_tier": score_tier,
            "total_issues": len(all_issues),
            "tier2_enabled": tier2_enabled,
            "short_circuited": False,
            "latency_ms": round(total_latency),
        },
    )

    return result
```

## Rubric Prompts (Full Text)

The three rubric prompts are defined as module-level constants in `llm_judge.py` (included in the code above). Key design principles:

1. **Numeric anchors with examples:** Each score range (0.0-0.2, 0.3-0.4, etc.) has a concrete description of what content at that level looks like. This eliminates the "what does 0.6 mean?" ambiguity.

2. **JSON output enforced:** We use OpenAI's `response_format={"type": "json_object"}` to guarantee valid JSON. The system prompt specifies the exact schema: `{"score": float, "reasoning": string}`.

3. **Domain-appropriate criteria:** The heading structure prompt checks SEO-specific things (keyword in H1, no skipped levels) rather than general writing quality. The brief adherence prompt includes the actual brief data.

4. **Temperature 0.0:** For evaluation (not generation), we want deterministic, reproducible scores. Temperature 0 gives the most consistent results.

5. **Truncation at 8000 chars:** Articles are typically 1000-2500 words (~5000-15000 chars). Truncating at 8000 chars (~2000 words) keeps input tokens reasonable while capturing enough content for evaluation.

## Error Handling & Graceful Degradation

### Principle: Tier 2 Failure Never Breaks the Pipeline

The entire Tier 2 block in `quality_pipeline.py` is wrapped in a try/except. Any failure mode results in the pipeline returning Tier 1 results with `tier2.error` set in the metadata.

### Failure Modes

| Failure | Handling | User Impact |
|---------|----------|-------------|
| `OPENAI_API_KEY` not set | `run_llm_judge_checks` returns immediately with error | Tier 1 score only, no LLM section in UI |
| `openai` package not installed | Import guard returns error | Same as above |
| OpenAI API timeout (>30s) | SDK raises `APITimeoutError`, caught in `score_*()` | That dimension returns `None`, others may succeed |
| OpenAI rate limit (429) | SDK auto-retries with backoff (built-in) | May succeed after retry, or timeout |
| OpenAI server error (500) | SDK auto-retries | Same as timeout |
| OpenAI auth failure (401) | Caught in `score_*()`, returns error | All 3 dimensions fail, Tier 1 score only |
| Malformed JSON response | `_parse_judge_response()` returns score=0.0 with error | That dimension treated as failed |
| Score out of range | Clamped to [0.0, 1.0] | No impact |
| `asyncio.gather` exception | `return_exceptions=True` prevents cancellation | Failed tasks return None, others succeed |

### Dev/Test Environment Handling

- **No OpenAI key in dev:** `QUALITY_TIER2_ENABLED=false` (default). Pipeline runs Tier 1 only. Zero impact.
- **Want to test Tier 2 locally:** Set `QUALITY_TIER2_ENABLED=true` and `OPENAI_API_KEY=sk-...` in `.env`. Pipeline runs full pipeline.
- **Tests:** All LLM calls mocked. No real API calls in test suite.

## Performance Considerations

### Latency Budget

```
Current (Tier 1 only):           <10ms
With Tier 2 (3 parallel calls):  +2-4s (median), +6-10s (p99)
Total pipeline:                  ~2-10s
```

This runs AFTER content is written (which takes 15-30s), so the additional 2-4s is acceptable. The `CHECKING` status is already shown in the UI during quality checks.

### Token Usage Per Article

```
Per judge call:
  Input:  ~3000-6000 tokens (system prompt ~800 + content ~2000-5000)
  Output: ~100-200 tokens (JSON with score + reasoning)

Per article (3 calls):
  Input:  ~9000-18000 tokens
  Output: ~300-600 tokens

Cost: $0.022-$0.045 per article (GPT-5.4 pricing)
```

### Batch Generation Impact

At `CONTENT_GENERATION_CONCURRENCY=1` (default):
- Pages processed sequentially
- 3 parallel OpenAI calls per page
- 50 pages = 150 calls, spread over ~50 * 20s = ~16 minutes
- Well within rate limits

At `CONTENT_GENERATION_CONCURRENCY=5`:
- 5 pages in parallel
- 15 parallel OpenAI calls at peak
- 50 pages = 150 calls, spread over ~3 minutes
- Still within rate limits (GPT-5.4 allows ~500 RPM on standard tier)

### Memory

No significant memory impact. Each judge call is a small HTTP request/response. No embedding storage, no model loading.

## Implementation Notes for the Developer

1. **Start with `quality_pipeline.py`** -- get the orchestrator working with Tier 2 disabled first. Verify all existing tests pass. Then add `llm_judge.py` and wire up Tier 2.

2. **The `content` parameter duality** -- For onboarding pages, `content` is a `PageContent` SQLAlchemy model that has `qa_results` as a JSONB column. For blog posts, `content` is `None` and we pass `fields` dict directly. The pipeline handles both via the `is_blog` flag.

3. **Import the pipeline lazily in content_generation.py** -- Since `quality_pipeline.py` imports from `content_quality.py`, and `content_generation.py` currently imports `run_quality_checks` from `content_quality.py`, there's no circular import risk. But the `run_llm_judge_checks` import inside `quality_pipeline.py` should remain at call-site (inside the function) to avoid importing openai at module load time.

4. **`flag_modified` for JSONB** -- When `run_quality_pipeline` mutates `content.qa_results` in-place, SQLAlchemy may not detect the change. The callers in `content_generation.py` already handle this for the existing `run_quality_checks`. Ensure the same `flag_modified(content, "qa_results")` pattern is used after the pipeline call.

5. **The `openai` SDK uses httpx under the hood** -- This means it participates in the same event loop as our existing httpx usage. No threading conflicts.

6. **Run `uv lock` after adding `openai` to dependencies** to regenerate the lockfile.
